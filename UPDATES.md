# تحديثات نظام راشد الاستخباراتي v2.1

## المشاكل التي تم حلها

### 1. مشكلة الرابط الفارغ على Render ✅

**المشكلة الأصلية:**
- عند الانتقال من Replit إلى Render، كان رابط التعقب يظهر صفحة فارغة
- السبب الرئيسي: تضارب المنافذ (Port Conflict) بسبب تشغيل خادمين Flask منفصلين

**الحل المطبق:**
1. **دمج خادمي Flask:** تم دمج `tracker_bot.py` و `keep_alive.py` في ملف واحد `tracker_server.py`
2. **تشغيل خيط منفصل:** الآن يعمل خادم التعقب في خيط منفصل (daemon thread) مع البوت الرئيسي
3. **تحديث Procfile:** تم تغيير نوع العملية من `worker` إلى `web` لتفعيل الرابط العام
4. **منفذ واحد:** كل الخدمات تعمل على منفذ واحد (PORT) المعرّف في Render

**الملفات المتأثرة:**
- `Procfile` - تم تحديثه
- `main_bot_updated.py` - ملف جديد يدمج كل شيء
- `tracker_server.py` - ملف جديد يحتوي على خادم التعقب

---

### 2. ربط الذكاء الاصطناعي بالإنترنت ✅

**المشكلة الأصلية:**
- الذكاء الاصطناعي (Groq) كان يعمل بدون اتصال بالإنترنت
- ميزة البحث (Tavily) كانت موجودة لكن لا تُستخدم تلقائياً

**الحل المطبق:**
1. **تفعيل البحث الحي:** تم تطوير دالة `_groq_post()` لتدعم البحث عبر الإنترنت
2. **دمج Tavily:** عند كل سؤال، يتم البحث عن معلومات حديثة وإضافتها للسياق
3. **تحسين الإجابات:** الآن الذكاء الاصطناعي يعطي إجابات مدعومة بمصادر من الإنترنت
4. **معلومات محدثة:** يمكن للبوت الإجابة عن أسئلة تتطلب معلومات حديثة

**الميزات الجديدة:**
```python
# البحث يحدث تلقائياً عند كل سؤال
reply = ask_groq(user_id, prompt, use_internet=True)

# يتم إضافة نتائج البحث للسياق
search_context = f"📌 *معلومات من البحث:*\n{answer}\n"
```

### 3. توحيد متغيرات البيئة ✅

**المشكلة الأصلية:**
- اختلاف بين أسماء المتغيرات في الكود والمثال
- `DEHASHED_API_KEY` في المثال بينما الكود يبحث عن `LEAKCHECK_KEY`
- `VIRUSTOTAL_KEY` في الكود بينما المثال يسميه `VIRUSTOTAL_API_KEY`

**الحل المطبق:**
1. **تحديث .env.example:** تم توحيد أسماء المتغيرات
2. **تحديث main_bot.py:** تم تعديل البحث عن المتغيرات
3. **توثيق واضح:** أسماء المتغيرات الآن متسقة في كل مكان

**المتغيرات الصحيحة:**
```env
GROQ_API_KEY=...
TAVILY_API_KEY=...
VIRUSTOTAL_API_KEY=...
LEAKCHECK_KEY=...
IPINFO_TOKEN=...
BOT_SERVER_URL=...
```

## خطوات التطبيق على Render

### 1. استبدال الملفات
```bash
# احذف الملفات القديمة
rm main_bot.py tracker_bot.py keep_alive.py run.sh

# انسخ الملفات الجديدة
cp main_bot_updated.py main_bot.py
cp tracker_server.py tracker_server.py
```

### 2. تحديث Procfile
```
web: python3 main_bot.py
```

### 3. تحديث متغيرات البيئة على Render
في لوحة تحكم Render، أضف أو حدّث:
- `MAIN_BOT_TOKEN`
- `TRACKER_BOT_TOKEN`
- `GROQ_API_KEY` ✅ (للذكاء الاصطناعي)
- `TAVILY_API_KEY` ✅ (للبحث عبر الإنترنت)
- `VIRUSTOTAL_API_KEY` (لفحص الروابط)
- `LEAKCHECK_KEY` (لفحص التسريبات)
- `IPINFO_TOKEN` (لتحليل الـ IP)
- `BOT_SERVER_URL` (سيتم ملؤه تلقائياً من `RENDER_EXTERNAL_URL`)

### 4. إعادة تشغيل الخدمة
- اذهب إلى Render Dashboard
- اختر الخدمة الخاصة بك
- اضغط "Manual Deploy" أو "Redeploy"
